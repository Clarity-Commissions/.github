# Security Policy

Clarity Commissions is a private SaaS platform. We take security seriously and appreciate responsible disclosure.

## Reporting a Security Issue
Please **do not** report security vulnerabilities via public GitHub issues, discussions, or pull requests.

Report security issues privately by emailing:
- **security@claritycommissions.com**

Include:
- A clear description of the issue and potential impact
- Steps to reproduce (proof-of-concept if available)
- Affected component/service/repository and environment details
- Any relevant logs, screenshots, or indicators of compromise (if applicable)
- Suggested remediation (optional)

If you need to share sensitive details, request a secure channel and we will coordinate encryption/file transfer.

## Response Expectations
We aim to:
- Acknowledge receipt within **3 business days**
- Provide an initial assessment within **7 business days**
- Work with you on remediation and, if needed, coordinated disclosure

Timelines may vary based on severity and complexity.

## Coordinated Disclosure
We support coordinated vulnerability disclosure and can provide reporter credit upon request (or honor anonymity).

Thank you for helping keep Clarity Commissions secure.
